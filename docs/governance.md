# 2. Политика логирования

Система Elyrion ведет журналирование основных операций для обеспечения наблюдаемости системы, отладки и анализа работы агентов.

## Что логируется

Логируются следующие события:

- пользовательские запросы к системе
- формирование агентом плана действий
- подтверждение или отклонение плана пользователем
- операции развертывания инфраструктуры
- результаты выполнения Terraform операций
- ошибки и исключения

## Что не логируется

В логах не должны сохраняться:

- токены доступа к GitHub
- API ключи
- секреты инфраструктуры
- персональные данные пользователей

## Уровни логирования

Используются стандартные уровни логирования:

- INFO — ключевые действия системы
- WARNING — потенциальные проблемы
- ERROR — ошибки выполнения операций
- DEBUG — подробные технические данные (используется только в режиме разработки)

---

# 3. Политика работы с персональными данными

В рамках PoC система Elyrion не предполагает хранение значительного объема персональных данных.

Система может хранить:

- идентификатор пользователя
- ссылку на GitHub-репозиторий
- метаданные развернутых проектов

Система не хранит:

- пароли пользователей
- содержимое репозиториев
- личные данные пользователей

При доступе к GitHub используется токен пользователя, который хранится в защищенном виде и используется только для получения доступа к указанному репозиторию.

---

# 4. Защита от Prompt Injection

Система предусматривает меры защиты от prompt injection атак, которые могут возникнуть при анализе содержимого GitHub-репозитория.

Меры защиты включают:

- использование фиксированных системных инструкций для LLM
- ограничение набора доступных инструментов агента
- проверку входных данных перед передачей в LLM
- игнорирование потенциально вредоносных инструкций из содержимого репозитория

LLM используется только для анализа и генерации плана действий.  
Критические операции выполняются детерминированным кодом.

---

# 5. Подтверждение критических действий

Все действия, которые могут изменить инфраструктуру, требуют явного подтверждения пользователя.

Процесс включает следующие этапы:

1. Агент анализирует проект пользователя.
2. Агент формирует план развертывания.
3. План отображается пользователю.
4. Пользователь подтверждает или отклоняет предложенный план.
5. Только после подтверждения выполняются инфраструктурные операции.

Этот механизм предотвращает выполнение нежелательных действий системой.

---

# 6. Ограничение полномочий агентов

Агенты системы Elyrion работают в ограниченной среде и не имеют прямого доступа к критическим системным ресурсам.

Ограничения включают:

- доступ только к разрешенным API
- выполнение инфраструктурных операций через Terraform
- отсутствие прямого доступа LLM к облачным провайдерам
- использование backend-логики для выполнения критических операций

Таким образом, агент выполняет аналитическую и планирующую роль, а исполнение инфраструктурных операций осуществляется контролируемыми сервисами системы.
